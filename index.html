Both tables have Auto Increment = 110

💡 Solution Requirements
• Ensure ALL services are updated before enabling Phase 2
• Add prerequisite checks in CI/CD pipeline
• Monitor for any remaining Phase 0 code in production
• Maintain sufficient buffer between auto-increment values
• Implement proper error handling for duplicate key conflicts
🔄 Phase 2 (Approach 2) - Use Temp Sequence Table
Generate sequence IDs from New Sequence Table and save to Temporary Sequence Table with buffer
Migration Strategy: Buffer + New-to-Temp Flow
Before migrating to Phase 2, we maintain a sufficient buffer in the new sequence table.
During Phase 2, we generate IDs from the new table and copy to the temporary table.
📊 Buffer Preparation (Before Phase 2)
Ensure new sequence table auto-increment is set to a value much higher than the old table

✅ Benefits of Approach 2
• Buffer prevents ID conflicts between old and new machines
• New table remains the primary source of truth
• Temporary table serves as a backup/copy
• Gradual migration without downtime
• Easy rollback to Phase 1 if needed

• No ID conflicts due to buffer gap (e.g., 200+ vs 110+)
• Stale machines continue dual-write normally
• New machines always use new table with higher IDs
• Both old and new tables remain synchronized
• Smooth transition without data loss
